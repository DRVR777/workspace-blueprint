# ADR-009: SOE Operates Independently but Shares RE and KBPM

## Status
accepted — stated explicitly in PRD Section 2.4

## Context
PRD states: "The SOE is a fully autonomous sub-system for trading Solana-native assets. It operates independently of the Polymarket modules but shares the same knowledge base, reasoning engine, and post-mortem system."

## Decision
The Solana Opportunistic Executor (SOE) is an independent program with its own entry/exit logic and statistical model. It does not depend on SIL, WADE, or OSFE output. However:
- It uses RE for AI floor estimates (calls RE as a service)
- All SOE trades are documented in KBPM (/markets/ and /theses/ vault)
- SOE post-mortems are generated by the same KBPM post-mortem pipeline as Polymarket trades

## Consequences
- SOE can run without Polymarket modules being active.
- SOE adds demand to RE — RE must handle concurrent requests from both Polymarket and SOE paths.
- SOE and Polymarket execution use separate TradeExecution records but share the same contract shape.
- Whether SOE uses the same execution wallet as Polymarket is resolved in ADR-021.
