# ADR-008: Every Decision Logged with Full Reasoning Context

## Status
accepted — stated explicitly in PRD Section 4

## Context
PRD states: "Every decision — executed or not — must be logged with full reasoning context. The operator must always be able to answer 'why did the system do that?'"

## Decision
Every TradeThesis generated by RE is logged to KBPM regardless of whether a trade is executed. Each log entry includes: the full context assembly, the hypotheses generated, the evidence weighting, the confidence score, the decision (execute / flag for review / skip), and if executed — the execution details.

The KBPM /theses/ vault is the audit trail. It is append-only.

## Consequences
- RE must pass full context to KBPM with every TradeThesis, not just the summary.
- KBPM must never delete thesis documents — only outcome labels are added post-resolution.
- The ThesisDocument shape in shared/contracts/ must carry enough context to reconstruct the reasoning.
- Storage grows continuously — vault size management strategy decided during KBPM build.
