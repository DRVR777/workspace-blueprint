# programs/context-builder — Task Router

## What This Program Is
Executes embedded prompts. Watches ticker.log for new entries. For each new read,
reads the file's neighbors via file-selector, then writes/updates ctx-NNNN.md.
This is the self-description layer — it makes documents teach the AI about themselves.

## The Context-Builder Loop

```
1. Read current ticker.log length (last seen position)
2. Poll ticker.log for new lines (every N seconds, default 2)
3. For each new line:
   a. Parse: file_number, session_id, reason
   b. Skip if reason is 'context_built' or 'indexed' (avoid loops)
   c. Skip if ctx-[file_number].md exists and is < staleness_threshold minutes old
   d. Read file[file_number].md to get: vector, neighbors, embedded prompt, content
   e. For each neighbor: call file-selector to read it (reason: 'neighbor_of_[N]')
   f. Execute embedded prompt logic to generate ctx content
   g. Write Data/ctx-[file_number].md
   h. Append to ticker.log: [timestamp] | [file_number] | [session] | context_built
4. Return to step 2
```

---

## What ctx-NNNN.md Contains

Generated by executing the file's embedded prompt:
1. **What I Am** — one paragraph summary of the file's content
2. **My Position** — 5D vector values interpreted in plain language
3. **My Neighbors** — how this file relates to each neighbor (one sentence each)
4. **My Cluster** — which topic area this file belongs to
5. **My Role** — "This document is a [noun] that [verb phrase]."

## Audit
- [ ] ctx-NNNN.md has all 5 required sections
- [ ] ctx file is NOT written if ctx exists and is < staleness_threshold old
- [ ] context_built entry in ticker.log appears after each ctx write
- [ ] Loop does not trigger on context_built or indexed ticker entries (infinite loop prevention)
- [ ] ctx file uses ONLY content from the file and its neighbors (no hallucination)
