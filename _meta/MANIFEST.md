## What I Am
Layer 0 of this workspace. Domain-agnostic infrastructure that operates above
the workspace layer. Contains the prompt generator (ur-prompt.md), the gap schema,
the gap detection agent, and the live gap registry.

This folder is also the canonical example of a complete MANIFEST. If you want to
understand how to write a MANIFEST for any other folder, read this one.

## What I Contain
| Name | Type | Purpose |
|------|------|---------|
| ur-prompt.md | file | Domain-agnostic orientation + execution prompt generator |
| gap-schema.json | file | Schema for all gap objects |
| gap-detection-agent.md | file | Agent prompt: converts inference logs → formal gap objects |
| runner.md | file | Session protocol: how to start, run, and end any session |
| prd-intake.md | file | Agent prompt: reads a PRD and scaffolds a complete project |
| spec-review.md | file | Gate agent: 4-check validation before scaffold → specced transition |
| status-transitions.md | file | Explicit gate criteria for every program and project status change |
| MANIFEST.md | file | This file — canonical MANIFEST example |
| gaps/ | folder | Live gap registry: pending.txt, open gap JSON files, CONTEXT.md index |
| intake-pipeline/ | folder | 5-stage document intake: envelope → classify → route → place → update |
| improvement-engine/ | folder | Self-improvement pipeline: telemetry → pattern detection → diff proposals → auto-apply or human-gate |
| graph-engine/ | folder | Knowledge graph materializer: crawls edge declarations → builds graph.json → detects orphans |
| guards/ | folder | Protective guards: finalized.flag cascade (Guard 3), MANIFEST staleness (Guard 5) |
| scripts/ | folder | Python automation scripts for project creation, gap detection, MANIFEST scaffolding |

## What I Need From Parent
Nothing. This folder is domain-agnostic. It requires no workspace-specific context.

## What I Give To Children
- gaps/ receives inference log entries from any workspace session

## What I Return To Parent
- Execution prompts (generated by ur-prompt.md from gap objects)
- Formal gap objects (generated by gap-detection-agent.md from pending.txt)
- These become the inputs for the next session's tasks

## Routing Rules
| Condition | Go To |
|-----------|-------|
| You need to orient in an unfamiliar workspace | ur-prompt.md |
| You need to generate a prompt for a new task | ur-prompt.md Steps 1–7 |
| You need to log an inference | gaps/pending.txt |
| You need to detect gaps from pending.txt | gap-detection-agent.md |
| You need to find the highest-priority open gap | gaps/CONTEXT.md |
| You have any inbound document | intake-pipeline/ stage-01-envelope |
| Run the improvement cycle | improvement-engine/CONTEXT.md |
| Build or rebuild the knowledge graph | graph-engine/CONTEXT.md |
| Check before writing to a finalized folder | guards/guard-03-finalized-flag.md |
| Check MANIFEST freshness at session end | guards/guard-05-manifest-reconcile.md |
| You have a PRD and need to scaffold a project (direct) | prd-intake.md |
| You need the session start/end protocol | runner.md |
| A program is ready to move from scaffold → specced | spec-review.md |
| You need to understand the gate criteria for any status change | status-transitions.md |
| Create a new project from a PRD | scripts/new_project.py or prd-intake.md |
| Check pending inference logs across all projects | scripts/run_gaps.py |

## Layer 0 Test
Every file in this folder must pass: "Would this work equally well for a legal
research system, a protein folding database, or Byzantine tax law?"
If any file mentions Acme, DevRel, blog posts, or demos — it has violated Layer 0
and must be moved to the appropriate workspace layer.
